## 1. 目的

バックテスト成績が「過去の特定タイミング・順序への依存（いわゆる運）」によって生まれたものか、それとも戦略ロジック自体の優位性から来る「再現性のある成績」なのかを統計的に検証する。

具体的に確認したい問い:
1. 日次リターンの順序をランダムに入れ替えても同等の成績が出るか？（**タイミング依存度の検証**）
2. Bull/Bear レジームの出現順序を変えても成績は安定するか？（**レジーム依存度の検証**）
3. 損失確率・テールリスク（CVaR）はどの程度か？
4. 実績 CAGR はシミュレーション分布の何パーセンタイルに位置するか？

## 3. Monte Carlo 手法

### 3-1. 手法 A: Stationary Block Bootstrap

**ライブラリ:** `arch.bootstrap.StationaryBootstrap`（`arch` パッケージ）

**原理:**
日次リターン系列を「可変長ブロック」単位でリサンプリングする。各ブロック長は幾何分布 Geom(1/b) から独立に抽選（期待値 = mean_block_size）。これにより:
- 短期の自己相関（ボラティリティクラスタリング）を保持
- IID リサンプリングよりも時系列の構造を保持

**パラメータ:**

| パラメータ | 値 | 根拠 |
|-----------|---|------|
| mean_block_size | 10 日 | 日次収益の自己相関が概ね 1〜2 週間で減衰。標準的な推奨値 (4th root of n ≈ 8〜9) に近い |
| N simulations | 2,000 | テール（P5）推定に十分な精度。1,000 より安定 |
| seed | 42 | 再現性確保 |

**実装コード（核心部分）:**
```python
from arch.bootstrap import StationaryBootstrap

bs = StationaryBootstrap(block_size=10, x=returns, seed=42)
for i, (pos_data, _) in enumerate(bs.bootstrap(2000)):
    sim_r = pos_data[0].flatten()[:len(returns)]
    eq    = np.cumprod(1 + np.clip(sim_r, -0.999999, None))
    # compute CAGR, Vol, MDD, Sharpe ...
```

**何をテストするか:**
- 取引の**順序・タイミング**がランダムだった場合に同等の成績が出るか
- 実績が「偶然の時系列配列に依存したもの」か「期待値として再現可能なもの」かを判定

---

### 3-2. 手法 B: Regime-Aware Bootstrap（レジーム認識ブートストラップ）

**原理:**
SMA160 シグナルで GSPC を **Bull** と **Bear** に分け、連続したレジームブロック（エピソード）単位でリサンプリングする。

**アルゴリズム:**
1. `is_bear[d] = GSPC_OPEN[d] < SMA160_PREV_CLOSE[d]` でレジームを識別
2. 連続する同一レジームの日数を「エピソード」として抽出 → `bull_episodes`, `bear_episodes` のリストを作成
3. ブートストラップパス生成:
   - 開始レジームを historical の最初に合わせる
   - Bull エピソード → Bear エピソード → Bull … を交互に置換抽出
   - 目標日数（= 元系列の長さ）に達するまで繰り返す

**実装コード（核心部分）:**
```python
def regime_bootstrap_path(bull_eps, bear_eps, target_len, rng, start_bear):
    path = []
    current_bear = start_bear
    while len(path) < target_len:
        eps = bear_eps if current_bear else bull_eps
        ep  = eps[rng.integers(len(eps))]      # 置換抽出
        path.extend(ep[:target_len - len(path)].tolist())
        current_bear = not current_bear
    return np.array(path[:target_len])
```

**何をテストするか:**
- Bull/Bear **レジームの出現順序** がランダムだった場合に成績は安定するか
- 「2007年の Bear の後に 2013年の Bull が来た」という歴史的偶然が成績に寄与しているかどうか

**手法 A との違い:**
- 手法 A: 日次リターンの「順序」をブロック単位でランダム化 → 個々のリターンの自己相関を保持しつつ長期構造を壊す
- 手法 B: レジームエピソード全体の「順序」をランダム化 → エピソード内の詳細な日次構造は完全に保持

## 4. 評価指標

| 指標 | 定義 | 意味 |
|------|------|------|
| **損失確率** | `mean(final_equity < 1.0)` | 全シム期間通じて元本割れするパスの割合 |
| **CVaR 5%** | 下位 5%パス の平均 CAGR | テールリスクの大きさ（Expected Shortfall） |
| **実績パーセンタイル順位** | 実績 CAGR がシミュレーション分布の何%ile か | **50th 付近 → 再現性高（タイミング依存低）。80th 超 → 運依存が疑われる** |
| **CAGR P5/P50/P95** | CAGR の 5/50/95 パーセンタイル | 成績の「幅」の把握 |
| **MDD P5/P50/P95** | 最大ドローダウンの分布 | テールシナリオのドローダウン推定 |

## 6. 解釈・考察

### 6-1. 主要知見

#### ✅ 知見①: 実績は「中央値」付近に位置する（全系列・全手法で一致）

| 系列 | 手法 A 実績順位 | 手法 B 実績順位 |
|------|---------------:|---------------:|
| from_20051220 | **51.8th** | **51.1th** |
| from_20100212 | **51.3th** | **47.5th** |
| 1991 stitched | **52.6th** | **48.6th** |

全系列・全手法で実績 CAGR は 47〜53rd パーセンタイルに収まっている。これは:
- 実績が「特別に運の良いシナリオ」ではなく、**標準的な経路で達成された再現性の高い成績**であることを意味する
- 戦略が過去の特定の順序・タイミングに大きく依存していないことを示す
- 一般的に「過剰最適化された戦略」は実績が 80th 以上に位置することが多い

#### ✅ 知見②: 損失確率ゼロ（全 6 パターン・2,000 シミュレーション）

8,887 日（35 年）の最長系列を含むすべてのシミュレーションにおいて、最終的に元本を下回ったパスは 0 件であった。TQQQ の長期的な上昇トレンドと SMA160 によるリスク管理の組み合わせが、リターンの順序をランダム化した場合でも元本保全を実現している。

#### ✅ 知見③: 手法 A と手法 B の結果が一致（P50 差 < 5pp）

| 系列 | A P50 | B P50 | 差 |
|------|------:|------:|---:|
| from_20051220 | 111.8% | 112.1% | 0.3pp |
| from_20100212 | 151.8% | 155.9% | 4.1pp |
| 1991 stitched | 67.8% | 69.2% | 1.4pp |

Bull/Bear レジームの出現**順序を入れ替えても**成績はほぼ変わらない。これは戦略が特定の歴史的レジーム配列（「2008年の暴落の後に 2009〜2020 年の大Bull相場が来た」等）に依存していないことを示す。

#### ⚠️ 知見④: 1991年起点は 2005年起点よりテールリスクが顕著に大きい

| 指標 | from_20051220 | from_20100212 | 1991 stitched |
|------|-------------:|-------------:|--------------:|
| CVaR 5%（手法 A） | 66.7% | 86.6% | 39.0% |
| MDD P50（中央値） | -58.5% | -61.7% | -74.8% |
| MDD P5（最悪域） | -74.7% | -78.5% | **-89.2%** |

1991年起点はドットコムバブル崩壊（2000〜2002 年）を含む。この期間、合成 TQQQ は最大 -99.9% の下落を記録した。SMA160 シグナルで一定量は回避できるが、MDD テールリスクは 2005年起点の約 1.2 倍に拡大する。

#### ⚠️ 知見⑤: pre-2005 合成系列の限界

| 課題 | 詳細 |
|------|------|
| ドローダウン alpha の変更 | 実カノニカルの alpha=94% を 100% に変更している。この変更がなければドットコム崩壊時に誤発動し CAGR ≈ -2%（元本ほぼ全損）になる。1991 起点で実戦するには alpha の再調整が必要。 |
| TQQQ 合成リターンの精度 | pre-2010 の `TQQQ_CC_RETURN_REBUILT` は NASDAQ-100 日次リターン × 3 × レバレッジ調整係数（calibrated）であり、実際の TQQQ とは乖離がある。 |
| CTO/OTC 分割なし | pre-2010 は close-to-close return のみ。実カノニカルが使用する「前日終値→当日始値（CTO）+ 当日始値→終値（OTC）」の 2 段階計算ができず、若干の乖離が生じる。 |
| wait_mix 代替の過剰単純化 | 実際の wait_mix（50%TMF + 50%GLD）は TMF・GLD の相関特性を持つが、^IRX Cash は無相関。バッドマーケット時の「逃避先としての価値」が過小評価されている可能性がある。 |

## 8. 総合評価

> **カノニカル LETF ローテーション戦略の Monte Carlo ロバストネス検証において、以下の 3 点が確認された:**
>
> 1. **実績は再現性が高い**: 全系列・全手法で実績 CAGR が中央値付近（47〜53rd percentile）に位置し、タイミング・レジーム順序への依存が低い
> 2. **損失リスクは限定的**: 35 年間の長期シミュレーション（2,000 パス）でも損失確率はゼロ
> 3. **テールリスクは系列によって大きく異なる**: 2010 年起点（CVaR5% ≈ 87〜89%）は頑健性が最も高い。ドットコム崩壊を含む 1991 年起点は CVaR5% が 34〜39% まで低下するため、長期リスクとして注意が必要
>
> **カノニカルをそのまま実運用するにあたっては、2005 年以降のデータで評価するのが最も現実的**（実際の TQQQ・UVIX・TMF・GLD が揃っている期間）。1991 年起点の結果はドットコム時代のストレステストとして参照する。
